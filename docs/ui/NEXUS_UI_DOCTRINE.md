# NEXUS UI DOCTRINE
## Operator-facing doctrine

This file is a derivative summary of `docs/ui/NEXUS_UI_MASTER_TRUTH.md`.
If any conflict exists, the master truth file wins.

## Core doctrine

Nexus UI is a hover-native ambient command OS.

The workspace is primary.
The system is predictive.
The chrome is minimal.
Everything important is available on intent.
Anything can be pinned.
The UI must adapt to the user.
The UI must remain operationally truthful.

## Canonical product feeling

Nexus should feel:
- calm
- personal
- predictive
- useful as a tool
- ahead of the user
- nearly empty at rest
- fully powerful on demand

## Canonical interaction rules

- Edge surfaces are hover-reveal by default.
- The chat bar is hover-reveal and recedes after work begins.
- Quick actions are adaptive and customizable.
- Major controls have keyboard parity.
- Opening state is curated, never generic.
- Static dashboard-first layouts are not canonical Nexus.
- No decorative pane survives without live utility.
- Local hover tools are preferred over persistent clutter.
- Users may pin any important surface.

## Canonical pane families

Adaptive open-state panes may include:
- Now
- What Changed
- In Motion
- Needs You
- Prepared For You
- Context

These are adaptive operational reveals, not permanent dashboard modules.

A release may not describe the UI as master-truth aligned unless the UI truth gate passes.
