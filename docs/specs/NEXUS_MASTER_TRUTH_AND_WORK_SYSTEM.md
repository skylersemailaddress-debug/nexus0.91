# Nexus Master Truth and Work System

Status: CANONICAL
Authority: Highest implementation truth for Nexus execution planning and completion tracking.

## Purpose
This file is the single governing execution truth for Nexus.
All roadmaps, checklists, gates, and subsystem plans must conform to this file.

## Top-level truth
Nexus is only considered complete when both of the following are true:

1. Nexus is a truthful stateful AI operating system.
2. Nexus is a truthful governed autonomous product company.

If either is false, Nexus is not 10/10 complete.

## Non-negotiable gates

### Gate A — Truthful AI OS
Must prove:
- continuity and resume are real
- memory changes behavior
- execution persists over time
- Builder creates useful capabilities
- UI surfaces are backed by live state
- release/runtime paths are stable
- claims are evidence-backed

### Gate B — Truthful autonomous product company
Must prove:
- external opportunities can be discovered
- opportunities can be ranked and selected
- products can be generated
- products can be wrapped for multiple surfaces
- launches can be prepared and measured
- economics can be tracked
- fleet maintenance works
- unsafe autonomous actions are blocked
- operator can inspect the full loop

## Completion states
Every subsystem must explicitly declare one of:
- NOT_STARTED
- STRUCTURAL_ONLY
- BOUNDED_PROTOTYPE
- OPERATIONAL
- RELEASE_CANDIDATE
- ENTERPRISE_READY

No subsystem may be described above its proven completion state.
